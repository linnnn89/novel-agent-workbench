# Codex 工作记录

## 2026-07-22：创作工作台 UI 第一阶段优化

### 目标

- 按已选“方案 1”把主窗口改造成作品树、正文编辑器、上下文检查器三栏工作台。
- 保留现有 Tkinter、业务服务、项目数据结构及 API Key 保存方式。
- 改善切换作品时的同步阻塞与编辑状态反馈。

### 关键变更

- 重构主窗口信息架构：左侧集中作品与章节，中间集中稿件编辑，右侧显示本章目标、上下文和模型服务。
- 将旧顶栏多组下拉菜单收拢为“功能”菜单；高频的生成、AI 审稿和确认稿件保持直接可见。
- 增加作品搜索、编辑字数、自动保存状态、模型配置状态、撤销/重做入口和初始最大化。
- 作品树改为展开时加载章节与草稿，避免启动时逐作品读取全部稿件。
- 切换作品改用 `project_state()` 轻量概览，不再自动执行包含仓库扫描的完整项目体检；完整体检仍保留在手动功能入口。
- 新增不依赖 GUI 环境的桌面辅助函数测试。

### 保持不变

- 未修改 API Key 的存储、掩码、读取或写入逻辑。
- 未新增、升级或卸载依赖。
- 未改变草稿、确认稿、审稿、精修及模型调用的业务接口。

### 验证

- `python -m py_compile src/novel_agent_workbench/desktop_app.py`：通过。
- `python -m unittest discover -s tests`：15 项通过。
- Windows 源码版窗口实际启动：通过，窗口响应正常。
- 使用方案 1 与最终窗口截图合并对照：三栏结构、主任务层级、主操作可见性和空状态均通过。

### 反证自审与限制

- 当前项目库为空，因此截图验证的是空状态；已填充作品后的长标题、深层章节树和超长模型名称尚未做真实数据截图回归。
- Tkinter 原生控件仍保留一定系统风格，无法达到定制组件框架的像素级圆角、图标和动效效果；本阶段优先完成布局、层级和响应反馈。
- 后续若迁移到 PySide6 或其他 UI 框架，会引入新依赖和较大重构，需要单独确认。

### USER_DECISION_REQUIRED

- 本阶段无。

## 2026-07-22：创作工作台 UI 第二阶段优化与 EXE 交付

### 目标

- 延续方案 1，把高频次级窗口统一到主工作台的视觉层级。
- 解决 Windows 125% 缩放下的弹窗裁切和主窗口操作区不可见问题。
- 在不改变 API Key 保存方式、不引入依赖的前提下，产出可直接运行的 Windows EXE。

### 关键变更

- 增加 Windows 进程级 DPI 感知，并统一次级窗口的创建、居中、标题区、正文区和底部操作区。
- 重整生成草稿、模型服务、AI 审稿提示、思考过程、大纲/章纲、世界观记录、记忆库、生成设置和规划条目等窗口。
- 将 AI 审稿和确认稿件固定在编辑器标题栏；保存、撤销、重做、重新生成和根据草稿精修集中到“功能 > 编辑”。
- 将字数、自动保存和模型配置状态统一放入顶栏，取消在高 DPI 下可能落到可视区外的底部工具栏。
- 新增 API Key 掩码回归测试，确认已保存的密钥引用不会被表单状态暴露。

### 保持不变

- 未修改 API Key 的保存、掩码、读取或写入逻辑。
- 未新增、升级或卸载依赖；构建使用现有 `.venv` 与 `-SkipInstall`。
- 未改变模型调用、项目结构和稿件业务接口。

### 验证

- 系统 Python 3.10：16 项单元测试通过。
- 项目 `.venv` Python 3.14：16 项单元测试通过。
- `python -m py_compile src/novel_agent_workbench/desktop_app.py`：通过。
- 生成草稿和模型服务弹窗在 Windows 125% 缩放下实机截图复验：通过，无裁切、重叠或不可见主按钮。
- 最终 EXE 构建与启动响应检查：通过；`dist/NovelAgentWorkbench/用户数据` 在构建时得到保留。
- 参考设计与最终 EXE 合并对照：通过，记录见 `design-qa.md`。

### 反证自审与限制

- 本次只验证了代表性的高频弹窗；低频窗口共享相同基础布局，但未逐个保留截图证据。

## 2026-07-22：创作工作台 UI 第三阶段反馈调整

### 目标

- 按用户反馈压缩右侧检查器，为正文编辑区释放宽度。
- 将“重新生成（随机）”和“根据审稿意见精修”提升为主界面快捷操作。
- 将低频模型服务配置从右栏移到顶栏，同时保持 API Key 机制不变。

### 关键变更

- 右栏最小宽度从 250 调整为 190，概览文本换行宽度同步收窄，内部入口改为纵向全宽按钮，避免控件自然宽度反向撑大侧栏。
- 正文标题区增加“重新生成（随机）”和“根据审稿意见精修”，并与 AI 审稿、确认稿件组成常驻动作区。
- 顶栏增加“模型设置”入口，右栏删除模型服务摘要和配置按钮；原“功能 > 模型”入口继续保留。
- 为避免小窗口下三栏自然宽度溢出，收紧作品搜索框和正文编辑器的初始请求宽度；实测总请求宽度从 1145 降到 1109，低于 1120 的窗口下限。
- 新增快捷动作可见性和检查器宽度关系测试。

### 保持不变

- API Key 的保存、掩码、读取与写入逻辑未修改。
- 重写、精修、AI 审稿和确认稿件继续调用原有业务方法。
- 未安装、升级或卸载依赖。

### 验证

- 系统 Python 3.10：18 项单元测试通过。
- 项目 `.venv` Python 3.14：18 项单元测试通过。
- `desktop_app.py` 编译检查：通过。
- PyInstaller `-SkipInstall` 构建：通过；用户数据目录保留。
- Windows 125% 缩放、1920 × 1080 物理像素实机截图：三栏完整，右栏约 200 px，四个稿件快捷按钮全部可见。
- 顶栏“模型设置”实际点击：成功打开原模型服务窗口，API Key 仍为掩码显示。
- 设计 QA：`design-qa.md` 最终结果为 `passed`。

### 反证自审与限制

- 右栏更窄后概览文字会增加换行，这是为正文空间做出的有意取舍。
- 真实超长作品名和多层章节树尚未作为本轮截图样本，但三栏最小请求宽度已经做程序级量测。
